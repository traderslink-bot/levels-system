## Crowded / Redundant Level Findings

The audit found examples where three or more levels were close enough that a trader may read them as one zone:

- ATER support: 0.54 / 0.55 / 0.5615
- ATER resistance: 1.10 / 1.12 / 1.14
- BIYA resistance: 1.59 / 1.61 / 1.65
- SAGT resistance: 2.39 / 2.43 / 2.47
- SKYQ support: 5.97 / 6.10 / 6.18 and sometimes 6.10 / 6.18 / 6.32
- XTLB resistance: 4.44 / 4.49 / 4.54
- ABTS support: 1.35 / 1.38 / 1.40 / 1.43
- VSME support: 0.90 / 0.9225 / 0.935

This does not always mean the engine is wrong. It means the Discord view may be too granular.

Recommended fix:

- Keep the raw levels internally.
- In Discord, collapse tight clusters into a range when three or more nearby levels sit inside a small percentage band.
- Example trader-facing style:
  - `Resistance: 2.39-2.47 zone (clustered, heavy)`
  - `Support: 0.90-0.935 zone (clustered)`
- Preserve the individual levels in operator audit artifacts.

## Wording Findings

The saved logs still include trader-facing phrases that should not be in Discord going forward:

- `LEVEL SNAPSHOT`
- `level map`
- `Status: Cleared`
- `Status: Lost`
- `Status: Testing`
- `Signal: ... severity | ... confidence`
- `Decision area`
- `alert direction move`
- `after the alert`
- `dip-buy`
- `price cleared X and is moving toward Y`
- `price lost X and is moving toward Y`

Important nuance: many of these logs were produced before the latest trader-language cleanup. Current tests already assert against several of these phrases, including `Status:`, `Signal:`, `Decision area`, `setup update`, `state recap`, `setup move`, `alert direction`, `after the alert`, `mapped`, and `price target`.

Recommended fix:

- Keep the current trader-language tests.
- Add a replay-style assertion over saved posts or fixture posts so future formatter regressions are caught against realistic full Discord bodies.
- Replace `moving toward` phrasing everywhere with neutral nearby-level language:
  - Better: `nearby resistance above is 4.28`
  - Avoid: `price is moving toward 4.28`

## Meaning / Story Findings

The saved runner report flagged several cases where a price sample crossed a known level but did not get a nearby clear/lost-support post. Examples:

- SAGT crossed several supports and resistances during fast movement.
- SEGG crossed support/resistance levels during quick down/up swings.
- SKYQ crossed resistance around 6.45 / 6.56 / 6.74 / 7.00 and support around 6.80 / 6.87.
- XTLB crossed several levels during its runner move and fade.

Some of these may be acceptable suppression during burst windows. Others may represent missed level-cross narration.

Recommended fix:

- Separate "missed level" logic from "intentionally suppressed because a stronger post already covered the move."
- Add a suppression reason that says a crossed level was covered by a higher-level breakout/failure post within the same move.
- If not covered, emit a single consolidated post rather than one post per crossed level.

Example:

```text
XTLB cleared a cluster around 3.56-3.75.
Next resistance above is 4.17-4.28.
Holding above 3.75 keeps the breakout attempt cleaner.
```

## Data Quality Findings

- daily insufficient/stale/missing recent candles
- 4h insufficient/suspicious gaps
- 5m stale/missing/incomplete current session data

This matters because some questionable gaps may be real chart structure, but some may be provider/cache artifacts from the temporary IBKR setup.

Recommended fix:

- Keep showing data-quality context only internally/operator-side.
- Use the post-fix level-quality audit before changing level logic for a single ticker.
- Re-run the same audit after switching away from IBKR to compare whether these gaps remain.
